EXPERIMENT 3 (Part A):

Disassemble the suspicious binary and analyze its code flow to determine what actions the malware is programmed to perform. Identify any suspicious API calls or functions that could indicate malicious intent.

Tool: Ghidra

Static malware analysis involves examining an executable without running it to understand its behavior. Disassembly and decompilation reveal program logic, API usage, and control flow, which help determine whether the binary performs malicious actions such as persistence, file manipulation, or network communication.

----------------------------------------
PART A: Creating the Suspicious Binary (Kali Linux VM)
----------------------------------------

Step 1: Install MinGW-w64
sudo apt update
sudo apt install mingw-w64

Step 2: Create Working Directory
mkdir ghidra_lab
cd ghidra_lab

Step 3: Create Suspicious Source Code
nano sample.c

#include <windows.h>
#include <stdio.h>

int main() {
    HKEY hKey;
    HANDLE hFile;
    DWORD bytesWritten;
    char data[] = "System initialized\n";

    // Create a hidden file (suspicious behavior)
    hFile = CreateFileA("C:\\Temp\\syslog.txt", GENERIC_WRITE, 0, NULL, CREATE_ALWAYS,
                        FILE_ATTRIBUTE_HIDDEN, NULL);

    WriteFile(hFile, data, sizeof(data), &bytesWritten, NULL);
    CloseHandle(hFile);

    // Add registry persistence
    RegCreateKeyExA(HKEY_CURRENT_USER,
        "Software\\Microsoft\\Windows\\CurrentVersion\\Run",
        0, NULL, 0, KEY_WRITE, NULL, &hKey, NULL);

    RegSetValueExA(hKey, "SystemUpdater", 0, REG_SZ,
        (BYTE*)"C:\\Temp\\sample.exe", 20);

    RegCloseKey(hKey);

    MessageBoxA(NULL, "Service Started", "System", MB_OK);

    return 0;
}

Save and exit.

Step 4: Compile Windows Executable
x86_64-w64-mingw32-gcc sample.c -o sample.exe

Step 5: Verify Binary Type
file sample.exe

Expected output:
PE32+ executable (console) x86-64, for MS Windows

Suspicious binary successfully created.

----------------------------------------
PART B: Preparing Ghidra (Kali Linux)
----------------------------------------

Install JDK:
sudo apt install -y openjdk-21-jdk

Install Ghidra:
sudo apt install -y ghidra

Launch Ghidra:
ghidra

Step 1: Create a New Project
- File → New Project
- Select Non-Shared Project
- Project name: Malware_Ghidra_Analysis
- Choose workspace directory

Step 2: Import the Suspicious Binary
- File → Import File
- Select sample.exe
- Format: PE
- Click OK

Step 3: Analyze the Binary
When prompted:
- Select Analyze
- Enable default analyzers
- Enable PE analyzers
- Enable Decompiler
- Click Analyze

----------------------------------------
PART C: Initial Binary Inspection
----------------------------------------

1. Symbol Tree (Functions & Imports)

Open:
Window → Symbol Tree

Shows:
- Functions
- Imported APIs
- Exported symbols
- Labels

Structure:
Symbol Tree
├── Functions
├── Imports
└── Labels

Check under Imports:
- CreateFileA
- WriteFile
- CloseHandle
- RegCreateKeyExA
- RegSetValueExA
- MessageBoxA

API Meaning:
- CreateFileA → File creation
- WriteFile → Writing data
- RegCreateKeyExA → Registry key creation
- RegSetValueExA → Registry persistence
- MessageBoxA → Popup message

----------------------------------------

2. Defined Strings

Open:
Window → Defined Strings

Shows:
- All text strings inside the executable

Expected strings:
- System initialized
- C:\Temp\syslog.txt
- Software\Microsoft\Windows\CurrentVersion\Run
- SystemUpdater
- C:\Temp\sample.exe
- Service Started
- System

Importance:
Strings reveal:
- File paths
- Registry keys
- Messages
- Commands
- URLs

----------------------------------------

3. Decompiler Window

Open:
Window → Decompiler

Shows C-like reconstructed code.

Example:
CreateFileA("C:\\Temp\\syslog.txt", ...);
WriteFile(...);
RegCreateKeyExA(...);
RegSetValueExA(...);
MessageBoxA(...);

Helps understand logic without assembly.

----------------------------------------

4. Listing Window (Assembly View)

Open:
Window → Listing

Shows assembly instructions.

Example:
PUSH offset "C:\\Temp\\syslog.txt"
CALL CreateFileA
CALL WriteFile
CALL RegCreateKeyExA

This represents actual machine execution.

----------------------------------------

RESULT
----------------------------------------

The suspicious binary was successfully disassembled using Ghidra. Analysis of its code flow and API calls revealed that it creates hidden files, establishes registry persistence, and executes automatically, indicating malicious intent.

----------------------------------------

CONCLUSION
----------------------------------------

Ghidra static analysis revealed that the analyzed binary performs unauthorized file system modification and registry persistence using Windows API calls. These behaviors are characteristic of malware designed to maintain persistence and operate stealthily on infected systems.
